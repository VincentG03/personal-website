SKILLS SECTION (Lines 7-44):
-----------------------------
There are 4 skill cards. Each has:
  - title: The skill name
  - description: What you do with this skill
  - icon: SVG graphic (you can keep as is or replace)

To change a skill:
1. Find the skill object (e.g., 'Full Stack Development')
2. Change the title property
3. Change the description property

Example at Line 9:
  {
    title: 'Full Stack Development',
    description: 'Experienced in building modern web applications with React, Node.js, and various databases.',

Change both title and description to your own skill.

Repeat for all 4 skills (lines 9-16, 17-24, 25-32, 33-40).


================================================================================
                         SECTION 3: WORK EXPERIENCE
================================================================================

FILE: components/ExperienceSection.tsx

Each job experience is an object in the "experiences" array starting at line 8.

There are 3 types of experience entries:
  A) Single role at a company
  B) Multiple roles at same company (promotions)
  C) Single role at another company


EXPERIENCE FORMAT A - SINGLE ROLE (Lines 8-25):
------------------------------------------------
Find:
  {
    role: 'Senior Software Engineer',
    company: 'Tech Innovations Inc.',
    period: 'Jan 2022',
    endPeriod: 'Present',
    logo: '/logos/tech-innovations.png',
    description: 'Leading development of cloud-native applications...',
    highlights: [
      'Reduced application load time by 60%...',
      'Led and mentored a team of 5 engineers...',
      'Implemented comprehensive CI/CD pipeline...',
    ],
  },

TO CHANGE:
- role: Your job title
- company: Company name
- period: Start date
- endPeriod: End date or "Present"
- description: Brief role description
- highlights: Array of achievements (add or remove bullet points as needed)


EXPERIENCE FORMAT B - MULTIPLE ROLES (Lines 26-73):
----------------------------------------------------
Used when you had promotions at the same company.

Find:
  {
    company: 'Growth Startup LLC',
    roles: [
      {
        role: 'Engineering Manager',
        period: 'Jun 2021',
        endPeriod: 'Dec 2021',
        description: '...',
        highlights: [...]
      },
      {
        role: 'Senior Developer',
        ...
      },
    ],
  },

TO CHANGE:
- company: Company name
- For each role inside roles array, update:
  - role: Job title
  - period: Start date
  - endPeriod: End date
  - description: What you did
  - highlights: Achievements


EXPERIENCE FORMAT C - ANOTHER SINGLE ROLE (Lines 74-85):
---------------------------------------------------------
Same as Format A above.


TO ADD NEW EXPERIENCE:
Copy an existing experience object (with all its properties) and paste it
into the array, then modify all the fields.

TO REMOVE EXPERIENCE:
Delete the entire object from the array, including the opening { and
closing }, and the comma after it.

================================================================================
                             TIPS AND NOTES
================================================================================

1. LINE NUMBERS: Line numbers may shift as you edit. Use the "Find" text
   to locate the exact section.

2. BACKUP: Always save a backup copy before making changes.

3. TESTING: After changes, run "npm run dev" to preview locally at
   http://localhost:3000

4. IMAGES: All images referenced as "/image.jpg" should be placed in the
   "public" folder at project root.

5. SPECIAL CHARACTERS: If you need apostrophes in text, use &apos; instead
   of ' to avoid errors (e.g., "Let&apos;s Connect" instead of "Let's Connect").

6. ARRAYS: When editing arrays (like skills, projects, experiences), always
   maintain proper commas between items. Last item should NOT have a comma.

7. DELETING SECTIONS: If you want to remove an entire section (like Societies),
   you need to:
   - Remove it from app/page.tsx (the import and component)
   - Remove it from the navbar in components/Navbar.tsx
